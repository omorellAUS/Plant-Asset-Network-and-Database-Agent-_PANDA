# Contributing to PANDA

Thank you for considering contributing to **PANDA** — Plant Asset and Network Database Agent.

This project is built as a truth-seeking knowledge bridge for maintenance and reliability. It must remain factual, transparent, and strictly focused on capturing execution reality without overstepping into automation or decision-making.

## Code of Conduct

- **Truth first**: All contributions must prioritize accuracy, evidence, and transparency over convenience or opinion.
- **Human judgement mandatory**: PANDA is a memory and evidence layer only. Never add features that automatically update systems, make decisions, or remove human review.
- **Blunt but respectful**: Code and comments should be clear and direct. No fluff, no moralising.
- **Low friction**: Keep field usage in mind — solutions should be simple and use tools people already have.

## How to Contribute

### 1. Reporting Issues
- Use the GitHub Issues tab.
- Clearly describe the problem with:
  - Expected behaviour
  - Actual behaviour
  - Steps to reproduce
  - Any relevant logs or screenshots

### 2. Suggesting Enhancements
- Open an issue with the label `enhancement`.
- Explain why the feature aligns with the Concept Brief (memory bridge, evidence over authority, low friction, human judgement mandatory).
- Include example outputs if possible.
